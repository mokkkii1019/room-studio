# LP 画像ガイド（運営者が用意する画像）

LP刷新で各LPに「画像スロット」を実装済み。**ファイルを `lp-assets/` に置くだけ**で自動的に表示されます。ファイルを置くと、その枠が自動で表示されます（before/afterは2枚そろったときだけ表示）。

- **ヒーロー(A)**: 写真が未配置の間は、**インラインSVGの室内イラスト**（明らかにイラストと分かる装飾。写真や「実例」を騙らない）を自動表示します。実写真ファイルを置くと自動でそちらに差し替わります。→ ヒーローは常に何かが表示され、空箱にはなりません。
- **before/after(B)**: 実キャプチャの真正性を守るため**画像のフォールバックはなし**（ダミー画像やフリー素材で代用しない）。未配置時の挙動はLPごとに2通り:
  - **北欧LP（`ph: True`）**: 「Before画像が入ります／…」と書かれた**破線のプレースホルダ枠**を表示。枠のサイズは実画像と同一なので、あとからファイルを置いてもレイアウトは動きません。
  - **他3LP**: 2枚そろうまで**セクションごと非表示**（従来どおり）。横展開したくなったら `api/_site.py` の該当LPに `"ph": True` を足すだけです。

- 置き場所: リポジトリ直下の **`lp-assets/`**
- 配信URL: `https://roomstudio.jp/lp-assets/<ファイル名>`（`/lp-assets/{name}` ルートが配信）
- **対応形式**: `.webp`（推奨）/ `.jpg` / `.jpeg` / `.png` / `.avif`。**拡張子は自動判別**（例 `hokuo-interior-hero.jpg` を置けば `hokuo-interior-hero` のスロットに入る）
- 置いたら `git add lp-assets/ && commit && push`（main反映で本番に出る）

## ⚠️ ライセンス注意（必読・厳守）

**要点2つ:** ①ヒーロー(A)のフリー素材ルール（下記）、②before/after(B)に**楽天収集家具を写さない**（さらに下）。

ヒーロー写真（下記A）は**フリー素材**を使います。必ず以下を守ってください。

- ✅ **商用利用可・帰属表示不要**の素材のみ（例: Unsplash / Pexels の標準ライセンス）。
- 🚫 **人物が写った写真は使わない**。モデルリリース（肖像の使用許諾）はフリー素材側で保証されません。**無人の室内・インテリアに限定**。
- 🚫 **ブランドロゴ・商標・他社商品が明確に写った写真は使わない**。
- 🚫 **Unsplash等のAPI経由で自動取得しない**。API利用は帰属表示が必須になる等、条件が変わります。**手動でダウンロードした静的ファイル**を置いてください。
- 🚫 フリー素材写真を「Room Studioで加工した実例」であるかのように見せない（それは下記Bの役割）。ヒーローは**あくまでイメージ写真**。

### 🔴 before/after に「収集した家具」を写してはいけない（楽天規約）

before/after（下記B）は Room Studio で作ってキャプチャしますが、**権利がクリアなのは
「あなた自身の部屋写真＋CC0の壁・床素材」だけを写した場合に限ります。**

**楽天から収集した家具を配置したキャプチャをLPに載せてはいけません。** LPは公開ページ＝
「不特定多数と共有可能な場所」であり、そこへ楽天の商品画像（を合成した画像）を置くことは、
楽天ウェブサービス利用規約に反します。当アプリの登録申請でも「目的外利用しない／再配布
しない／不特定多数と共有可能な場所へ保管しない」と明記しており（`rakuten-application-info.md`）、
LP掲載はこのいずれにも抵触します。画像は `/imgproxy`・`/bgcut` とも保存しない方針です。

before/after のAfterに家具を写したい場合は、**楽天以外の権利がクリアな家具**に限ります:

- ✅ あなたが撮影した／所有している家具の写真をアップロードして配置
- ✅ CC0・商用可ライセンスの家具画像
- ✅ **壁・床の素材（CC0のambientCG）や配色だけを変えた**キャプチャ（家具を置かない）
- 🚫 「家具を収集」機能で楽天から取り込んだ家具を配置したキャプチャ
- 🚫 どうしても楽天商品を使うなら、楽天ウェブサービス／該当店舗から**書面の許諾**が必要

なお下記 §3(賃貸の壁)・§4(北欧)のAfterは**壁・床の変更のみ**の指示なので、そのとおりに
作ればクリア。§1・§2は家具を伴うため、上記の「楽天以外の家具」で作ること。

## 画像の役割（2種類）

- **A ＝ ヒーロー写真**（フリー素材の無人インテリア）… ページ最上部の「引き」。世界観づくり。
- **B ＝ before/after キャプチャ**（Room Studioの実画面）… 「実際にこう変えられる」の証明。**各LPの主役**。

## 各LPに置くファイル一覧

推奨: ヒーロー=**16:9・横1600px前後・WebPで300KB以下**、before/after=**4:3・横800px前後・各150KB以下**。

| スロット | 比率 | 推奨実寸 | 形式 | 上限 | 読み込み |
|---|---|---|---|---|---|
| `<slug>-hero` | 16:9 | 1600×900 px | webp推奨 / jpg・png・avif可 | 300KB | `eager`（ファーストビュー。LCP対策） |
| `<slug>-before` | 4:3 | 800×600 px | 同上 | 150KB | `lazy` |
| `<slug>-after` | 4:3 | 800×600 px | 同上 | 150KB | `lazy` |

alt と `width`/`height` はコード側で付与済み（CLSが出ないよう実寸を明示しています）。
**before と after は必ず同じ比率**にしてください（横並びが破綻します）。上の推奨px通りに書き出せば揃います。

## 配置のコツ
- WebP変換とリサイズは [Squooss](https://squoosh.app/) 等で。横1600px・品質75前後で十分きれいかつ軽量。
- before/after は**同じ画角・同じ部屋**で撮ると対比が伝わります（Room Studioの「比較」機能でスクショしてもOK。ただし**楽天収集家具は写り込ませない**——上の🔴）。
- alt文言はコード側に実装済みなので、ファイルを置くだけで差し替わります（変更したい場合は `api/_site.py` の各LPの `hero`/`ba` の `alt` を編集）。
