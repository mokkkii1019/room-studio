# LP強化（FAQ・内部リンク・機能訴求）実装レポート

既存4枚のLPに、SEO・回遊・成約を強化する4要素（FAQ＋FAQPage構造化データ／LP相互内部リンク／家電・雑貨の言及／手持ち家具の撮影・配置の言及）を追加。既存トーン・捏造なし・実機能準拠・UGC非訴求を維持。

- ブランチ: `feature/lp-enhance`（`feature/lp-copy` から分岐＝LP本文の上に重ねる）
- 変更ファイル: `api/_site.py` のみ（＋ `docs/LP_ENHANCE_PLAN.md` / 本レポート）

---

## 1. 実装内容

### (a) FAQ ＋ FAQPage 構造化データ
- 各LPの本文3節の後・CTAの手前に「よくある質問」を追加（`<h2>` ＋ 各設問 `<section class="faq"><h3>Q</h3><p>A</p>`）。
- 各LPに **FAQPage の JSON-LD** を追加（既存の WebPage JSON-LD とは別の2つ目の `ld+json` script。どちらも有効）。
- 設問数: 6畳=2 / ソファ=3 / 賃貸=2 / 北欧=2（指示どおり）。

### (b) 内部リンク（関連ページ）
- FAQの後・フッターの前に「関連ページ」ブロック。アンカーテキストは**リンク先のH1**（自然な日本語）。
- 接続（§2どおり）: 6畳→ソファ/北欧/賃貸、ソファ→6畳/北欧、賃貸→6畳/北欧、北欧→ソファ/6畳/賃貸。自己リンクなし・全リンク先200。

### (c) 家電・インテリア雑貨の言及（独立見出しなし・文中に自然に）
- 6畳: ③に「ラグ・カーテン・フロアランプの色みも6畳の見え方を左右」。
- ソファ: ③に「ラグ・クッション・フロアランプの色をそろえるとまとまる」。
- 北欧: ③に「観葉植物・アート・あたたかみのある照明で北欧らしさが増す」。
- 賃貸: 壁が主役のため家電は足さず、③で「カーテンやアートの色みまで含めて見比べる」に留めた（§3-3の方針）。

### (d) 手持ち家具の撮影・配置の言及（各LP1文・私的利用の文脈）
- 6畳②「いま使っているテレビや収納を写真に撮って背景を切り抜けば…」／ソファ②「いま使っているテレビ台やローテーブルを…背景を切り抜き…」（背景切り抜きに軽く言及）。
- 賃貸②「いま置いている家具を写真で一緒に並べれば、家具と新しい壁色の相性も…」。
- 北欧②「いま持っている家具を写真に撮って置き、北欧テイストに馴染むかを…」。
- **公開・共有・投稿には一切言及なし**（§0-4遵守）。

### レンダリング（`landing_html`）
- `faq` / `related` はLP定義の任意フィールド。FAQ・関連リンク・FAQPage JSON-LDは**存在するときだけ**出力（後方互換）。
- CSS（`h3` / `.faq` / `.related`）を追加。既存のメタ（title/description/canonical/OGP）・CTA・法務フッターは不変。

### 賃貸LPのFAQ（§3-3・慎重表現を厳守／本文抜粋）
> Q. 貼ってはがせる壁紙なら、賃貸でも必ず元に戻せますか？
> A. 「はがせる」とされる壁紙でも、貼る期間や下地の状態によっては、はがす際に既存の壁紙を傷めたり、のり跡が残ったりすることがあります。原状回復できるかは製品や物件によって異なるため、購入前に商品説明を確認し、心配な場合は目立たない場所で試すのがおすすめです。賃貸借契約書の原状回復の取り決めも確認しておくと安心です。

「必ず元に戻せる」と読める断定は無し。注意喚起＋契約確認の案内を保持（むしろ強化）。

## 4. 仮定と判断

- **ブランチ**: §6の選択肢のうち「`feature/lp-enhance` を `feature/lp-copy` から分岐」を採用（本文と拡張を分離）。**マージ依存あり**（下記§6）。
- **FAQ構造化データ**: WebPageと`@graph`で束ねず、**独立した2つ目の `ld+json`**で追加（実装が単純で、どちらも Google に有効）。
- **関連リンクのアンカー**: §1-2「H1に近い自然な日本語」に沿い、**リンク先のH1そのもの**を使用。
- **家電/雑貨**: テーマに合うものだけ文中に溶かし、賃貸は家電を足さず雑貨（カーテン/アート）に留めた（§1-3・§3-3）。
- **背景切り抜き**: 「自動/AI/手動」を列挙せず「背景を切り抜けば」と軽く言及（§1-4「主役にしない」を優先）。
- **見出しの微修正**: 雑貨を加えた③の見出しを「〜の色や雑貨でも/で」等に自然調整（トーン・意味は不変）。

## 6. 残課題・申し送り

- **本番反映の手順（マージ依存）**: `feature/lp-enhance` は `feature/lp-copy` を含む。
  - 推奨: `feature/lp-copy` のPRを先にmainへマージ → その後 `feature/lp-enhance` のPRを出すと差分がFAQ/内部リンク/機能訴求のみになりレビューしやすい。
  - あるいは `feature/lp-enhance` を直接mainへマージすれば、**本文＋強化を一括反映**（lp-copyのPRは不要になる）。どちらを取るかは運営判断。
- **Google の FAQ リッチリザルト**は現在（2023以降）表示対象が限定的だが、FAQPage構造化データ自体は有効・無害でSEO上のマイナスはない。
- **LP増設・ディープリンク**は従来どおり将来拡張（`LANDING_PAGES` に追加すればFAQ/内部リンク/sitemapの仕組みに自動追従）。
