# LP before/after 枠 + ヒーロー差し替え — 計画

対象: 北欧LP `/lp/hokuo-interior` のみ（他3枚は今回対象外）
ブランチ: `feature/lp-beforeafter`（main へ直接コミットしない）

## 0. 着手時の実地調査（重要）

指示書の要求の大半は **すでに実装済み** だった。本番 `/lp/hokuo-interior` の
レンダリング結果で確認:

| 指示書の要求 | 着手前の状態 |
|---|---|
| §1 before/after 対比枠（2列→縦積み・ラベル・キャプション・同一比率） | 実装済み（`_site.py` の `ba` 設定 + `.ba-grid` / `.ba-label` / `.ba-cap`） |
| §2 ヒーロー差し替え構造（実ファイル優先・無ければイラスト） | 実装済み（`_img(slug-hero)` else `_hero_svg()`） |
| §3 画像仕様書 | 実装済み（`docs/LP_IMAGE_GUIDE.md` にサイズ・比率・形式・内容指示・置き場所・ライセンス注意） |
| CLS 防止 | 実装済み（`.ph` の `aspect-ratio` で領域確保） |

本番HTMLの実測: `class="ba"` = 0件（画像未配置のため非表示）、`hero-ph illus` = 1件
（SVGフォールバック表示中）、`lp-assets` 参照 = 0件。つまり**枠は完成しており
`lp-assets/` が空なだけ**。

したがって本作業の実体は「ゼロから実装」ではなく、下記の**差分の解消**である。

## 1. 指示書と現状の食い違い（と判断）

### (a) 未配置時にプレースホルダを出すか → **出す（運営判断で確定）**

指示書 §0-3 は「未配置でもプレースホルダ表示」を要求。しかし直近コミット
`efe8927`「未配置の画像スロットは公開ページで非表示に（クリーンな空状態）」で
**逆の判断**が下されており、`LP_IMAGE_GUIDE.md` にも「実キャプチャの真正性を守るため
フォールバックなし。2枚そろうまで非表示」と明記されていた。

公開中の本番サイトに開発用テキストが出る変更のため運営に確認 → **指示書通り常時表示に戻す**
との回答。よって `efe8927` の判断を北欧LPに限って覆す。

- [x] 判断を記録（本節）。`LP_IMAGE_GUIDE.md` の該当記述も更新すること。

### (b) 共有レンダラなので「北欧LPのみ」に限定する必要がある

`landing_html()` は4枚のLPを同じコードで描画する。素直に実装すると4枚すべてに
プレースホルダが出て §4-2「北欧LPのみ」「他3枚は未変更」に反する。

→ **LPデータに `ph: True` フラグを持たせ、北欧LPだけ True にする。**
他3枚はフラグ無し＝従来通り「2枚そろうまで非表示」。横展開時はフラグを足すだけ。

### (c) 置き場所は指示書の `/static/lp/hokuo/` ではなく既存の `lp-assets/` を採用

指示書は `/static/lp/hokuo/hero.webp` を例示するが、既存実装は `lp-assets/<slug>-<role>`
（拡張子自動判別）で `/lp-assets/{name}` ルートが配信済み。既存規約を壊す理由がないため
**既存の `lp-assets/` を維持**し、指示書の例示は読み替える。`LP_IMAGE_GUIDE.md` に既出。

### (d) 「各画像にキャプション欄」の解釈

現状は各画像に Before/After **ラベル**＋2枚共通の説明文（`.ba-cap`）。
指示書 §1 は「各画像にキャプション欄を設ける」。
→ データに既存の `note_b` / `note_a`（「編集前：ふつうの部屋」「編集後：床と壁を北欧トーンに」）
を**各画像のキャプションとして描画**し、共通の `.ba-cap` は既存デザインのまま残す。
新規文言を発明せず、リデザイン済みの構成も壊さない。

### (e) ヒーローの `loading`

`_img()` が全スロットに `loading="lazy"` を固定付与しており、ヒーロー写真を置いた
瞬間に LCP が悪化する。指示書 §0-5・§2 も「ヒーローは eager」。
→ `_img()` に `eager` 引数を追加し、ヒーローのみ `loading="eager"` + `fetchpriority="high"`。

- [x] 運営承認済み（「修正する」）。

### (f) 明示 width/height

指示書 §0-5 は明示 `width`/`height` を要求。現状は `aspect-ratio` のみ（CLS は防げている）。
→ 二重の保険として `<img>` にも `width`/`height` を付与（hero 1600x900 / ba 800x600）。
